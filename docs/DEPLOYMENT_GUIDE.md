# Инструкции по развертыванию

## Локальная сборка и тестирование

### Шаг 1: Клонирование репозитория
```bash
git clone https://github.com/your-repo/Alphearea.git
cd Alphearea
```

### Шаг 2: Установка зависимостей
```bash
npm install
```

### Шаг 3: Запуск локального dev-server
```bash
npm run dev
```

### Шаг 4: Проверка работы заглушек API
```bash
node testApiMocks.js
```

### Шаг 5: Проверка SPA-маршрутизации и стилей
Откройте браузер и перейдите по адресу `http://localhost:3000`. Убедитесь, что все страницы и стили работают корректно.

## Развертывание на VPS

### Шаг 1: Сборка проекта для production
```bash
npm run build
```

### Шаг 2: Загрузка файлов на сервер
Используйте `scp` или другой метод для загрузки файлов на сервер:
```bash
scp -r dist/ user@your-vps-domain.com:/path/to/project
```

### Шаг 3: Запуск сервера
```bash
serve -s dist -l 3000 --single
```

### Шаг 4: Настройка заголовков безопасности
Убедитесь, что на сервере настроены следующие заголовки:
- `Content-Security-Policy`
- `X-Content-Type-Options`
- `X-Frame-Options`

### Шаг 5: Настройка домена и SSL
Настройте домен и SSL-сертификат для вашего сайта. Например, с помощью `certbot`:
```bash
sudo certbot --nginx -d your-vps-domain.com
```

## Проверка результата

### Шаг 1: Проверка работы сайта
Откройте браузер и перейдите по адресу `https://your-vps-domain.com`. Убедитесь, что все страницы и функции работают корректно.

### Шаг 2: Проверка работы API
Убедитесь, что все API вызовы работают корректно и возвращают ожидаемые результаты.

### Шаг 3: Проверка безопасности
Проверьте, что все заголовки безопасности настроены корректно и сайт защищен от атак.

## Полезные команды

- **Запуск локального сервера**: `npm run dev`
- **Сборка проекта**: `npm run build`
- **Запуск тестов**: `npm test`
- **Проверка заглушек**: `node testApiMocks.js`
- **Запуск сервера для production**: `serve -s dist -l 3000 --single`
