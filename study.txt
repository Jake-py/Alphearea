- Какие коды и языки используются на сайте:
  - Языки программирования: JavaScript (основной для логики), HTML (структура страниц), CSS (стилизация).
  - Фронтенд: React (библиотека для создания пользовательского интерфейса), с использованием JSX для компонентов. Сборка через Vite.
  - Бэкенд: Node.js с Express.js для создания сервера и API.
  - Дополнительно: ES6 модули, JSON для данных.

- Библиотеки и библиотеки для ИИ какие используются:
  - Фронтенд библиотеки: React, React DOM, React Router DOM (для маршрутизации страниц), mammoth (для чтения .docx файлов), pdf-parse (для парсинга PDF файлов), iconv-lite (для обработки различных кодировок текста).
  - Бэкенд библиотеки: Express (веб-сервер), cors (для кросс-доменных запросов), axios (для HTTP запросов), argon2 (для хэширования паролей).
  - Для ИИ: Используется Ollama с моделью Gemma 2 (gemma2:2b) для генерации ответов в чате. Запросы отправляются на локальный сервер Ollama (по умолчанию http://localhost:11434/api/generate).

- Какие файлы есть и за что они отвечают:
  - Фронтенд (src/):
    - App.jsx: Главный компонент приложения, управляет маршрутизацией, аутентификацией, состоянием чата и сайдбара.
    - main.jsx: Точка входа, рендерит App в DOM.
    - components/: Компоненты UI, такие как Header (шапка), Sidebar (боковое меню), ChatPanel (панель чата с ИИ), SmartMaterialViewer (просмотр материалов), MaterialTreeView (дерево материалов).
    - pages/: Страницы приложения, например English.jsx (главная английского), EnglishGrammar.jsx (грамматика), Settings.jsx (настройки), и т.д.
    - styles/: CSS файлы для стилизации, например style.css, ChatPanel.css.
    - utils/: Утилиты, например scanMaterials.js (скрипт для сканирования и индексации материалов в JSON).
    - data/materials/: Материалы для обучения (тексты, PDF), с materials.json (индекс всех материалов).
  - Бэкенд (back-end/):
    - server.js: Основной сервер, обрабатывает API эндпоинты для чата, аутентификации, профилей, хэширования паролей.
    - config/siteInfo.js: Конфигурация сайта (имя, описание, FAQ).
    - accounts/: Директория для хранения данных аккаунтов (JSON файлы).
    - profiles/: Директория для профилей пользователей.
  - Корневые файлы: package.json (зависимости фронтенда), back-end/package.json (зависимости бэкенда), index.html (главная HTML страница), vite.config.js (конфиг Vite).

- Что за тема Argon2id:
  - Argon2id - это алгоритм хэширования паролей, используемый для безопасного хранения паролей. Он является гибридом Argon2i (для защиты от side-channel атак) и Argon2d (для защиты от GPU-атак). В коде используется для хэширования паролей пользователей при регистрации и верификации при логине. Параметры: memoryCost=64MB, timeCost=3, parallelism=1, hashLength=32.

- Зачем нужна тема и на что отвечают back-end и front-end:
  - Argon2id нужна для защиты паролей: хэширует их так, чтобы нельзя было легко восстановить оригинал, даже если база данных скомпрометирована. Это предотвращает brute-force атаки и rainbow table атаки.
  - Front-end: Отвечает за пользовательский интерфейс - отображение страниц, взаимодействие с пользователем (формы логина, чат, просмотр материалов), отправка запросов на бэкенд. Использует React для динамичного обновления UI.
  - Back-end: Отвечает за серверную логику - обработка API запросов (чат с ИИ, аутентификация, управление профилями), хранение данных (аккаунты, профили), интеграция с ИИ (Ollama). Обеспечивает безопасность (хэширование паролей, rate limiting) и бизнес-логику.
